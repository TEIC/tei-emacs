The first steps with Jade's TeX backend

Before we actually try out the TeX backend, we should take the time to perform a small test with our MiKTeX installation. Without MiKTeX being set up properly, the TeX backend will be useless anyway. You do not have to learn TeX or LaTeX to perform the test, simply follow the instructions below.

Start a new TeX file in your Emacs by typing C-x C-f textest.tex. Enter or paste the following lines:

\documentclass[letter]{article}

\begin{document}

\section{X, 34}
For one bitten by true doctrines even the briefest and most familiar saying is reminder enough to dispel sorrow and fear, for instance:

\begin{verse}
Like as the generation of leaves, even such are the children of men.\\
The wind scatters them on the face of the ground, but others the woodland\\
Brings forth again in its strength and they shoot in the season of spring;\\
Like to them are the children of men, one waxes, another is waning.
\end{verse}


\end{document}

Now save the file by typing C-x C-s. Open a command-line window and change to the directory where you saved your textest.tex file. Now enter the command:

~# latex textest.tex

This should create the files textest.aux, textest.log, and textest.dvi, which are an auxiliary file, a log file, and the formatted document, respectively. View the output by typing:

~# yap textest.dvi &

This should open Yap and load the output file. You should see a nicely formatted poem now.

But now back to SGML. Change to the directory that contains the test.sgml document that you created previously, and run this command:

~# openjade -t tex -d test.dsl test.sgml

This will create the file test.tex. This is a LaTeX file which needs some special macros (the JadeTeX stuff) to be interpreted correctly by TeX. So the command line is:

~# jadetex test.tex

This should create the standard set of .aux, .log, and .dvi files. View test.dvi with Yap. This should somewhat resemble the RTF output that you created from the same SGML source.

Alternatively, try to directly create a PDF document by typing:

~# pdfjadetex test.tex

View the PDF file by typing:

~# gsview test.pdf &

Finally you can try to create a PostScript document from your test.dvi. To this end, type the command:

~# dvips test.dvi

and view the resulting file with:

~# gsview32 test.ps &
