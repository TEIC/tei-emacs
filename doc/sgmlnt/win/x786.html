Install MiKTeX

  1. Unpack all MiKTeX archives into the same (temporary) directory. This will extract the installer, setupwiz.exe as well as a whole bunch of .zip archives (do not unpack them, setupwiz.exe will do this on the fly). Start the setup program setupwiz.exe. This will run a wizard which guides you through the installation.

    • Use the "Install a shared MiKTeX environment" option

    • Select a suitable installation directory like c:\Programs\texmf

    • In the following listbox, leave all components selected unless you really know what you're doing

    • Use whatever program folder pleases you

    • The wizard will then ask you for a local texmf tree (see above). Instead of the default which contains a space in the path (this will sooner or later cause problems) you should enter something like c:\Programs\localtexmf

    • When asked for existing texmf trees, select "Don't incorporate" unless you have such a resource, e.g. on the TeX Live CD or on a mounted network drive

    • Accept the suggestion to let the installer add the MiKTeX bin directory to your PATH (unless you are paranoid, then you'll have to do this yourself)

  2. Refresh the PostScript resource database

    Some utilities like Yap (the MiKTeX DVI previewer) need information about installed PostScript resources. This information is stored in the PostScript resource database.

    The following command collects the information about the MiKTeX PostScript resources as well as searches for external PostScript fonts (e.g. those included with Ghostscript):

    C:\Programs\texmf\>initexmf --mkpsres --search

  3. Configure DVIPS

    You'll probably have to change some settings for Dvips for your local site. The settings are stored in the file config.ps in the \Texmf\dvips\config directory. The following options might need a change:

    • The line starting with M should be modified according to the make and model of the printer that you use. The file \metafont\misc\modes.mf contains a list of possible values. Remember that a printer which is not in the list may be happy with the mode of a related printer, e.g. most recent HP LaserJet printers work with the ljet4 setting.

    • The line starting with D specifies the resolution of your printer. This always means the physical resolution, not what some resolution enhancement technology claims to make of the latter.

    • The line starting with O (the capital letter, not zero) specifies the horizontal and vertical printer offsets. These are printer-specific values which can be used to adjust the printout on the paper. The easiest way to determine these values is to use the file testpage.tex, which you will find in the \texmf\tex\latex\base\ directory. Open a command-line window and change into this directory. Type latex testpage.tex and answer the questions that appear on your screen. A testpage.dvi will be created. Use dvips -o testpage.ps testpage.dvi to convert this to a PostScript file. Now print this file on your printer using GhostView. You will see a box which should be evenly spaced with a 1 inch margin on all sides. The rulers help you to calculate which offsets (if at all) have to be specified in config.ps to adjust the printout correctly. The offsets can be specified as cm, as in, or as pt. In addition, these rulers show you the printer-specific unprintable area on the edges.

  4. Configure Yap

    Use the View->Options menu command to open the configuration dialog box. In the Display and in the Printer tab, select the default printer on your local system. In the Inverse Search tab, select "GNU Emacs (Single Instance)" from the drop-down list to register Emacs (via gnuclientw, to be precise) as the editor of choice.
